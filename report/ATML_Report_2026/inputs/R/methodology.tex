\subsection{Methodology}
This replication was conducted as an independent re-implementation from the paper's description, without access to the original authors' codebase. All experiments were re-written from scratch in Python using publicly available model checkpoints, libraries, and datasets. The sections reproduced are Section 3.3 (object discovery), Section 3.4 (qualitative evaluation), Appendix C (analysis of LOST performance), and Appendix D (behaviour of models trained with registers). Code was run on a single NVIDIA GPU, a single/double T4 (on Kaggle), with model loading handled via the \texttt{timm} and \texttt{open\_clip} libraries and, where necessary, HuggingFace \texttt{transformers}.

Three deviation from the paper are noted:
\textit{(1) Per-row mean-centering:} without centering, DINOv2 keys produce an all-positive Gram matrix causing degree-uniformity collapse; we subtract the per-row mean from G after bias addition. This suppresses genuine similarity variation and is the primary cause of the systematic 15-20\% CorLoc gap with the paper's numbers.
\textit{(2) Outlier threshold:} the paper uses a fixed threshold of 150 for DINOv2-g; we use Otsu's method on the pre-LayerNorm norm histogram (with fallbacks to median+10×MAD and the 95th percentile), since the fixed value does not generalise across model sizes.
\textit{(3) Position prediction:} we coarsen the paper's $37 \times 37$ position grid (1,369 classes) to $7 \times 7$ (49 classes) and subsample to 50k/20k train/test patches to make logistic regression tractable; this lowers absolute accuracy but leaves the relative model comparison unaffected.

\subsubsection{Experimental setup and code}
All experiments use PyTorch forward hooks to intercept intermediate model activations without modifying the underlying model architecture. This was the central technical pattern throughout the replication.

\textbf{Object discovery: LOST quantitative evaluation (Table~\ref{tab:object_discovery}).}
A hook on \texttt{model.blocks[-1].attn.qkv} (\texttt{timm}) or
\texttt{resblocks[-1].attn} (\texttt{OpenCLIP}) captures the raw QKV output;
keys or values are separated by reshaping the \texttt{(B, N, 3C)} tensor.
Special tokens (CLS + registers) are stripped by index.
The remaining patch features pass through:
L2 normalise $\rightarrow$
compute $G = \hat{K}\hat{K}^{\top} + b$,
subtract per-row mean $\rightarrow$
binarise at $\tau = 0$ $\rightarrow$
compute patch degrees $\rightarrow$
$\arg\min$ degree = seed $\rightarrow$
expand to all patches with similarity $> 0$.
The tightest bounding box around the expanded set is rescaled to original image size
and scored against all GT boxes; an image is counted correct if
$\max \mathrm{IoU} \geq 0.5$.
Inference runs in FP16 on GPU (\texttt{batch size = 32});
LOST runs per-image on CPU in float32. Code is available in \texttt{object\_discovery\_table\_3.py} \\
\textit{Evaluation metric:} CorLoc — \% images with IoU(predicted box, best GT box) $\geq$ 0.5.

\textbf{Qualitative attention maps (Figure~\ref{fig:qualitative_attention}).}
A hook on \texttt{model.blocks[-1].attn}
recomputes $\mathrm{softmax}(QK^{\top}/\sqrt{d})$
from the QKV output.
The resulting \texttt{(B, H, N, N)} matrix is averaged across heads;
rows for CLS and each register are reshaped to a $37 \times 37$ spatial grid.
Code is available in \texttt{dino\_att\_maps\_comparison.py}

\textbf{LOST intermediate maps (Figure~\ref{fig:lost_intermediate_steps}).}
Same hook and pipeline as above on a single image.
Three maps are retained and plotted as spatial grids:
(1) inverse-degree ($1/\mathrm{degree}$, high = candidate foreground),
(2) raw similarity row of the seed patch,
(3) binary seed-expansion mask.
For Figure~\ref{fig:seed_expansion_openclip} (\texttt{OpenCLIP} values),
the hook is moved to the full
\texttt{ResidualAttentionBlock} output instead of the QKV projection ---
the pre-LN normalisation in \texttt{OpenCLIP}
erases norm differences between patches when projecting directly to $V$. Code is available in \texttt{appendix\_C.py} \\
\textit{Evaluation metric:} qualitative assessment of the three LOST intermediate maps.

\textbf{Norm distributions (Figure~\ref{fig:token_norms_dinov2}).}
A hook on \texttt{model.norm} captures \texttt{inputs[0]}
(pre-LayerNorm residual stream), not output ---
post-LayerNorm activations are rescaled by learned parameters
and destroy the bimodal norm signal.
L2 norms are computed per token;
tokens are split by type using index positions
(CLS at 0, registers at 1--4, patches at 5--end)
and plotted as jittered strip plots over 500 COCO images. Code is available in \texttt{appendix\_D1.py} \\
\textit{Evaluation metric:} visual comparison of norm strip plots.

\textbf{Information held by tokens (Table ~\ref{tab:outlier_token_probes} and Table ~\ref{tab:local_information_probes}).}
A hook on the last encoder block
(\texttt{model.encoder.layer[-1]}, HuggingFace)
captures post-LayerNorm features.
The same block's pre-LayerNorm input is used to compute patch norms
for outlier detection (Otsu threshold).
Three probes are trained per token type:
(1) logistic regression on FGVC-Aircraft (global classification),
(2) logistic regression on a $7 \times 7$ position grid
(local position prediction),
(3) Ridge regression from embedding to raw pixels (reconstruction).
All features are standardised with \texttt{StandardScaler} before training. Code is available in \texttt{appendix\_D2.py} \\
\textit{Evaluation metric:} top-1 accuracy on FGVC-Aircraft; top-1 position prediction over 49 classes; mean L2 pixel reconstruction error.

\textbf{Positional focus (Figure~\ref{fig:positional_focus}).}
\texttt{fused\_attn=False} is set on all blocks
to materialise the softmax matrix.
A hook on \texttt{model.blocks[-1].attn.attn\_drop}
captures the
\texttt{(B, H, N\_total, N\_total)}
softmax weight tensor.
Averaged across $H = 16$ heads per image,
rows for CLS (index 0),
each register (indices 1--4),
and the centre patch (index $5 + 685$)
are accumulated over 500 Oxford-IIIT Pet images
and reshaped to $37 \times 37$ grids. Code is available in \texttt{appendix\_D3.py} \\
\textit{Evaluation metric:} visual comparison of attention heatmaps.
