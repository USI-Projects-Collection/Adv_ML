\subsection{Methodology}
This replication was conducted as an independent re-implementation from the paper's description, without access to the original authors' codebase. All experiments were re-written from scratch in Python using publicly available model checkpoints, libraries, and datasets. The sections reproduced are Section 3.3 (object discovery), Section 3.4 (qualitative evaluation), Appendix C (analysis of LOST performance), and Appendix D (behaviour of models trained with registers). Code was run on a single NVIDIA GPU, a single/double T4 (on Kaggle), with model loading handled via the \texttt{timm} and \texttt{open\_clip} libraries and, where necessary, HuggingFace \texttt{transformers}.

\textbf{Outlier threshold (TODO: Link the relative experiment to this outlier threshold (Appendix D.1, D.2))}
The paper reports a fixed threshold of 150 for identifying high-norm outlier patches in DINOv2-g. Rather than using a fixed value — which fails to generalise across model sizes due to different embedding scales — we implemented an adaptive threshold using Otsu's method applied to the pre-LayerNorm patch norm histogram, clipped at the 99th percentile to prevent the outlier tail from distorting the histogram. A sanity check ensures detected outliers comprise 1-15\% of patches; if this fails, the method falls back to $\text{median} + 10 \times \text{MAD}$ (median absolute deviation), and finally to the 95th percentile as a hard fallback. This approach is more principled than the paper's fixed value and generalises across model sizes.

\textbf{Position prediction approximation (TODO: Link the relative experiment to this position prediction (Appendix D.2, Paper Table 5))}
The paper predicts the exact patch grid position ($37 \times 37 = 1,369 \text{classes}$). We coarsen this to a $7 \times 7 = 49\text{-class grid}$ (each super-cell covering approximately $5 \times 5$ fine patches) to make logistic regression tractable within reasonable time. We also subsample to 50,000 training patches and 20,000 test patches. These are implementation approximations whose effect is visible in the absolute accuracy numbers but not in the relative trend between models, which is the quantity the paper reports.
