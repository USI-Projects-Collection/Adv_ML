\subsection{Methodology}

We reproduce Section~3.2 of \citet{darcet2024vision}, which studies whether adding register tokens to a pre-trained ViT degrades or improves downstream linear evaluation. We use three frozen backbones in a no-register and a with-register variant: \textbf{DINOv2 ViT-L/14} (the cleanest comparison, since both the baseline and the 4-register checkpoint are released directly by the authors via \texttt{timm} \citep{rw2019timm}); \textbf{OpenCLIP ViT-B/16} \citep{ilharco_gabriel_2021_5143773} (no trained-register checkpoint is publicly available, so we substitute the test-time register variant of \citet{jiang2025vision}); and \textbf{DeiT-III ViT-B/16} (no public register checkpoint, so we inject 4 untrained register-token \texttt{nn.Parameter}s at inference). Both substitutions are documented limitations.

We evaluate on three downstream tasks: (i) \textbf{Top-1 ImageNet classification} via logistic regression on frozen CLS features; (ii) \textbf{ADE20k mIoU} via a linear $1\!\times\!1$ \texttt{Conv2d} on patch features, bilinearly upsampled to mask resolution; and (iii) \textbf{NYUd RMSE} via the same single-channel linear head on log-depth. Due to compute constraints (Apple M-series MPS, no dedicated GPU) we use stratified subsets of 500 images each, split 7:3. Absolute numbers therefore will not match the paper; we compare only the \emph{relative} effect of adding registers. For zero-shot classification (Table~2b), we use the 11-prompt template ensemble in OpenCLIP's joint embedding space.

For the Figure~8 ablation over $N \in \{0,1,2,4,8,16\}$ register tokens, we compare three approaches. (1)~\textbf{Truncate/cycle}: since we cannot retrain, we use DINOv2-reg4 as a proxy and truncate its trained registers to the first $N$ for $N<4$, or cycle them for $N>4$. This produces near-binary behaviour (the model expects exactly 4 registers) rather than a smooth curve. (2)~\textbf{Test-time registers} \citep{jiang2025vision}: we re-implement Algorithm~1 (\textsc{FindRegisterNeurons}) from scratch on the \texttt{timm} DINOv2 port (GELU MLP, as opposed to the authors' SwiGLU target), identify 50 register neurons on 100 training images, and inject $N$ extra register tokens at inference by redirecting those neurons' activations -- no retraining required. Five of our top-10 neurons matched the indices published by \citet{jiang2025vision} for the SwiGLU architecture, validating the re-implementation. (3)~\textbf{Independent variant}: Jiang's intervention duplicates the top-1 outlier value identically across all $N$ registers. We test whether this duplication causes the performance collapse we observed at $N=16$ by distributing the top-$N$ distinct outlier values across $N$ separate registers instead. All code is in \texttt{Adv\_ML/src/Poli/}.
